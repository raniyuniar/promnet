GIT TOPIC

Version Control System (VCS)

Version Control System (VCS) adalah sebuah sistem yang digunakan dalam pengembangan perangkat lunak untuk melacak perubahan dalam kode sumber dan dokumen proyek sepanjang waktu. Tujuan utama dari VCS adalah untuk mengelola versi berbeda dari file dan mengkoordinasikan kolaborasi antara tim pengembang yang bekerja pada proyek yang sama.

Pengertian GIT

GIT adalah sistem kontrol versi yang populer dalam pengembangan perangkat lunak, yang memungkinkan pengembang untuk melacak dan mengelola perubahan kode sumber secara efisien dalam proyek. Ini membantu dalam kolaborasi tim, mencatat riwayat perubahan, dan memfasilitasi manajemen versi perangkat lunak.

Perbedaan Git dan Github

Git

  • Git adalah sistem kontrol versi distribusi yang digunakan untuk melacak perubahan dalam kode sumber proyek perangkat lunak.
  • Git berfokus pada fungsi-fungsi dasar kontrol versi, seperti merekam perubahan, menciptakan cabang (branch), melakukan merge, dan lainnya.
  • Git berjalan pada komputer lokal pengembang dan tidak memerlukan koneksi internet untuk berfungsi.
  • Git tidak menyediakan fasilitas hosting untuk repository kode sumber. Pengembang biasanya harus membuat repository Git di komputer mereka sendiri atau menggunakan server Git yang tersedia.

Github

  • GitHub adalah platform web berbasis Git yang menyediakan layanan hosting untuk repositori Git.
  • GitHub menyediakan fitur-fitur kolaborasi, seperti Pull Requests (permintaan gabungan), Issues (masalah), dan Wiki, yang memungkinkan tim pengembang untuk berkolaborasi dengan lebih efisien.
  • GitHub menyimpan repositori Git di servernya sendiri, yang membuatnya dapat diakses dari mana saja dengan koneksi internet.
  • GitHub menyediakan alat-alat untuk manajemen proyek, melacak masalah, dan dokumentasi.

Git Bassic Command

  • git init: Inisialisasi repository Git baru di direktori saat ini. Ini membuat repository kosong dan memungkinkan Anda memulai melacak perubahan kode.
  • git add [nama file]: Menambahkan file atau perubahan ke "staging area", yang merupakan persiapan untuk commit. Misalnya, git add file.txt akan memasukkan file.txt ke dalam staging area.
  • git commit -m "[pesan commit]": Membuat commit dengan pesan deskriptif yang menjelaskan perubahan yang telah Anda lakukan.
  • git status: Menampilkan status file dalam repository Anda, seperti file yang diubah, yang sudah di-add, atau yang belum di-add.
  • git pull: Mengambil perubahan terbaru dari repository yang sudah ada (biasanya dari server jarak jauh) dan menggabungkannya dengan branch saat ini.
  • git push: Mengirim perubahan lokal ke repository yang sudah ada (biasanya ke server jarak jauh) untuk menyimpan perubahan tersebut.
  • git branch: Menampilkan daftar branch yang ada dalam repository, dengan branch saat ini dicetak tebal.
  • git checkout [nama branch]: Beralih ke branch lain. Misalnya, git checkout develop akan beralih ke branch "develop."
  • git remote: Menampilkan daftar remote repository yang terhubung ke repository lokal Anda.
